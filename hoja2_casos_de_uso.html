Hoja 2 — Casos de Uso

Sistema Analizado: Plataforma de Banca Digital

En esta sección se identifican y documentan los principales casos de uso del sistema de banca digital. Cada caso de uso describe una interacción específica entre los actores del sistema y la plataforma con el objetivo de cumplir una funcionalidad del negocio.
Definición UML: Un caso de uso describe una interacción entre un actor y el sistema para lograr un objetivo específico. Cada caso de uso incluye un identificador, actor asociado, descripción funcional, precondición y postcondición.
ID Caso de Uso Actor Descripción Precondición Postcondición
CU-01 Iniciar sesión Cliente El cliente accede a la plataforma ingresando sus credenciales para autenticarse en el sistema. El cliente posee una cuenta registrada. El usuario accede al panel principal.
CU-02 Consultar saldo Cliente El cliente visualiza el saldo disponible de sus cuentas bancarias. El cliente ha iniciado sesión. Se muestran los saldos actualizados.
CU-03 Consultar historial de transacciones Cliente El cliente revisa el historial de movimientos de sus cuentas. El cliente tiene al menos una cuenta activa. Se muestra el listado de transacciones.
CU-04 Transferir fondos entre cuentas propias Cliente El cliente transfiere dinero entre sus propias cuentas registradas en el banco. El cliente posee saldo suficiente. Se actualizan los saldos de ambas cuentas.
CU-05 Transferir fondos a otro cliente Cliente El cliente envía dinero a otro cliente del mismo banco. La cuenta destino existe. Se registra la transferencia.
CU-06 Pagar servicios públicos Cliente El cliente paga servicios como electricidad, agua o internet mediante la plataforma. El cliente tiene saldo suficiente. El sistema registra el pago.
CU-07 Gestionar usuarios Administrador El administrador consulta y modifica información de usuarios. Administrador autenticado. Datos del usuario actualizados.
CU-08 Bloquear o habilitar cuentas Administrador El administrador controla el estado de las cuentas de clientes. Cuenta registrada en el sistema. Estado de la cuenta actualizado.
CU-09 Supervisar operaciones Administrador El administrador monitorea transacciones realizadas en el sistema. Administrador autenticado. Registro de auditoría actualizado.
CU-10 Procesar pago de servicios Proveedor de Servicios El proveedor valida y procesa la solicitud de pago enviada por el sistema. El cliente inicia el pago. Pago confirmado o rechazado.
