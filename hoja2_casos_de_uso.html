Hoja 2 — Casos de Uso

Sistema: Aplicación Digital Bancaria — Análisis y Diseño de Sistemas

Definición: Un caso de uso describe una interacción específica entre un actor y el sistema para lograr un objetivo concreto. Cada caso de uso tiene un nombre, actor(es) involucrado(s), precondición y postcondición.
👤 Cliente Casos de uso — Actor: Cliente
ID Caso de Uso Descripción Precondición Postcondición
CU-01
Iniciar sesión
Cliente		 El cliente accede al sistema ingresando sus credenciales (usuario y contraseña) de forma segura para autenticarse en la plataforma. El cliente tiene una cuenta registrada y credenciales válidas. El cliente accede al panel principal del sistema.
CU-02
Consultar saldo
Cliente		 El cliente visualiza el saldo disponible y/o total de una o más de sus cuentas bancarias en tiempo real. El cliente ha iniciado sesión exitosamente. Se muestra el saldo actualizado de la cuenta seleccionada.
CU-03
Revisar historial de transacciones
Cliente		 El cliente consulta el registro cronológico de movimientos (débitos, créditos, transferencias) realizados en sus cuentas. El cliente ha iniciado sesión y posee al menos una cuenta activa. Se despliega el historial de movimientos filtrable por fecha.
CU-04
Transferir dinero entre cuentas propias
Cliente		 El cliente transfiere fondos entre sus propias cuentas registradas dentro del banco. El cliente tiene al menos dos cuentas activas y saldo suficiente. Los saldos de ambas cuentas se actualizan y se genera un comprobante.
CU-05
Transferir a otros clientes del banco
Cliente		 El cliente envía fondos a la cuenta de otro cliente registrado en el mismo banco. Cuenta de destino existe y está activa; cliente tiene saldo suficiente. Se debita el monto de la cuenta origen y se acredita en la cuenta destino.
CU-06
Pagar servicios básicos
Cliente Empresa		 El cliente realiza el pago de servicios públicos (luz, agua, telefonía, internet) directamente desde la aplicación. El sistema interactúa con la empresa proveedora para validar y procesar el pago. El cliente tiene saldo suficiente y la empresa proveedora está integrada al sistema. El pago queda registrado y la empresa proveedora confirma la transacción.
🛡️ Administrador Casos de uso — Actor: Administrador del Sistema
ID Caso de Uso Descripción Precondición Postcondición
CU-07
Gestionar información de usuarios
Admin		 El administrador puede ver, editar y actualizar la información personal y bancaria de los usuarios registrados en la plataforma. El administrador ha iniciado sesión con privilegios de administración. La información del usuario queda actualizada en el sistema.
CU-08
Habilitar / bloquear cuentas
Admin		 El administrador activa o suspende cuentas de clientes según criterios de seguridad, sospecha de fraude o solicitud del cliente. La cuenta del usuario existe en el sistema. El estado de la cuenta cambia a activo o bloqueado según la acción.
CU-09
Supervisar operaciones de la plataforma
Admin		 El administrador monitorea en tiempo real todas las transacciones y operaciones realizadas para garantizar la seguridad y el cumplimiento de las políticas internas del banco. El administrador está autenticado con privilegios de supervisión. Se genera un registro de auditoría de las operaciones revisadas.
🏢 Empresa Externa Casos de uso — Actor: Empresa Proveedora
ID Caso de Uso Descripción Precondición Postcondición
CU-10
Validar y procesar pago de servicios
Externo		 La empresa proveedora recibe la solicitud de pago enviada por el sistema bancario, valida la referencia o deuda del cliente y confirma o rechaza el procesamiento del pago. El cliente ha iniciado el pago de un servicio (CU-06) y el sistema envía la solicitud a la empresa. La empresa confirma el pago; el sistema registra la transacción como completada o fallida.
